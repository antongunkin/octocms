# OctoCMS ŌĆö Content Management for AI Agents

This document describes how to manage content in an OctoCMS project directly via file operations.
Use it when an AI agent needs to create, read, update, or delete content entries without the admin UI.

For **this project's** collection list, field definitions, and URL mapping, see the auto-generated docs in `cms/__generated__/agent-docs/` (regenerated by `npm run agent-docs:gen`).

## How content is stored

Content lives in the project's `contentFolder` (typically `cms/content/`) as JSON files. Each collection has its own subfolder.

**File naming:**

- **`hasMany` collections** (multiple entries): `{contentFolder}/{type}/{type}-{uuid}.json` ŌĆö UUID is generated via `crypto.randomUUID()`
- **Singleton collections** (single entry): `{contentFolder}/{type}/{type}-0000.json` ŌĆö fixed ID `0000`

## Where the schema lives

The data model is defined in **`cms/schema.json`** ŌĆö the source of truth. It is plain JSON, hand-editable, and the visual Content Model editor at `/cms/model` writes to the same file.

`cms/octocms.config.ts` is a thin TypeScript binding that re-exports `cms/__generated__/schema.ts` (a literal-typed mirror of `cms/schema.json` regenerated by `npm run types:gen`). Editing `cms/octocms.config.ts` directly is **not** supported ŌĆö edit `cms/schema.json` instead.

See [editing-schema.md](./editing-schema.md) for how to edit `cms/schema.json` safely (validation invariants, regeneration commands, and migration semantics).

## Entry JSON structure

Every entry file has this shape:

```json
{
  "sys": {
    "id": "<uuid-or-fixed-id>",
    "type": "<collectionName>",
    "status": "draft"
  },
  "fields": {
    "fieldName": "value"
  }
}
```

## Entry status values

| Status | Meaning | Visible on public site |
| --- | --- | --- |
| `draft` | Newly created, never published | No |
| `published` | Live on public site | Yes |
| `changed` | Edited after publish, not yet re-published | Yes (shows latest) |
| `merged` | On main branch, baseline state | Yes |
| `archived` | Soft-deleted | No |

When creating a new entry, set `"status": "draft"`.

## Companion files (markdown & richtext)

Fields with `format: "markdown"` or `format: "richtext"` are **not** stored in the JSON entry.
Instead, they use companion files alongside the entry:

- **Markdown:** `{type}-{id}.{fieldName}.md`
- **Richtext:** `{type}-{id}.{fieldName}.mdx`

Example: A post entry `post-abc123.json` with a `body` markdown field stores its content in `post-abc123.body.md` in the same directory.

See `cms/__generated__/agent-docs/collections.md` for which collections in your project use companion files.

## Creating a new entry

1. Generate a UUID (e.g. `crypto.randomUUID()`)
2. Create `{contentFolder}/{type}/{type}-{uuid}.json` with the entry JSON structure above
3. Set `sys.status` to `"draft"`
4. Fill in all required fields (see `cms/__generated__/agent-docs/schema.md`)
5. For any markdown/richtext fields, create the companion file in the same directory

## Updating an entry

1. Read the JSON file
2. Modify values in the `fields` object
3. Write the file back
4. For markdown/richtext fields, edit the companion `.md`/`.mdx` file directly

## Deleting an entry

1. Delete the JSON file
2. Delete any companion files (`{type}-{id}.*.md`, `{type}-{id}.*.mdx`)

## Adding a new collection

1. Edit `cms/schema.json` ŌĆö add the new collection key under `collections` (see [editing-schema.md](./editing-schema.md) for the JSON shape and validation rules)
2. Create the content directory: `{contentFolder}/{collectionName}/`
3. Run `npm run types:gen` to regenerate `cms/__generated__/` (TypeScript types, query helper, literal-typed schema shim)
4. Run `npm run agent-docs:gen` to refresh `cms/__generated__/agent-docs/` (or rely on `types:gen`, which includes the same files)
5. Run `npm run docs:gen` to refresh `docs/generated/`
6. Optionally create a public page route using the `query()` API:

```typescript
import { query } from 'cms/__generated__/query';

const entries = await query('collectionName')
  .sort('fieldName', 'desc')
  .toArray();
```
