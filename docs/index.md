# OctoCMS — AI Agent Documentation

OctoCMS splits AI agent docs into two layers:

1. **Package docs** (`octocms/docs/`) — content-agnostic guidance shipped with the npm package. Stable across projects.
2. **Project docs** (`cms/__generated__/agent-docs/`) — auto-generated from your project's `cms/schema.json`. Regenerated by `npm run agent-docs:gen`.

Include both in your `AGENTS.md` or reference them directly.

## Package docs (stable)

- **[Content Management](./overview.md)** — Generic CRUD, entry shape, status values, companion files
- **[Editing the Schema](./editing-schema.md)** — How to edit `cms/schema.json` safely

## Project docs (auto-generated)

After running `npm run types:gen`, read these from your project root:

- **`cms/__generated__/agent-docs/schema.md`** — Per-collection field definitions and example JSON
- **`cms/__generated__/agent-docs/collections.md`** — Collection list, companion-file paths, URL mapping
- **`cms/__generated__/agent-docs/index.md`** — Project index with links to both layers

## Usage in AGENTS.md

```markdown
## Content Management

See `octocms/docs/overview.md` for generic content operations.
See `cms/__generated__/agent-docs/schema.md` for this project's field definitions.
See `cms/__generated__/agent-docs/collections.md` for collection and URL mapping.
See `octocms/docs/editing-schema.md` for schema editing.
```

Run `npx octocms agent-docs` to inject this section into `AGENTS.md`.

To regenerate after schema changes: `npm run agent-docs:gen`
